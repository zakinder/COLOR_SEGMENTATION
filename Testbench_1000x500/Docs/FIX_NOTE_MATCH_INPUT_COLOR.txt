Fix Note: Match Input Color Mode
================================

Root cause of previous mismatch:
  1. rgb_hsv.vhd had ENABLE_FLAT_COLOR=true.
  2. HUE_STEP=64, SATURATION_STEP=128, VALUE_STEP=128 forced heavy posterization.
  3. LOW_SATURATION_TO_GRAY=true forced low-saturation areas to gray.
  4. The testbench had PREFILTER_MODE=3, applying 3x3 median + 3x3 box blur before HSV.

This updated package changes the default behavior to color-preserving round trip:
  RGB input -> rgb_hsv -> hsv2rgb -> RGB output

Updated defaults:
  ENABLE_FLAT_COLOR       = false
  LOW_SATURATION_TO_GRAY  = false
  LOW_VALUE_TO_BLACK      = false
  PREFILTER_MODE          = 0

Expected behavior:
  Output visually matches the input image.
  Small +/- few-count differences can remain due to integer HSV conversion rounding.
